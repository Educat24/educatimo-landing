# План внедрения ТЗ: обновление контента лендинга (позиционирование B2B)

**Цель:** Смена позиционирования с «инструмент для педагога» на «бизнес-инструмент для владельца».  
**Охват:** Все главные страницы локалей (RU, UK, EN, PL, CZ).  
**Ограничения:** Не менять другие разделы, не трогать логику отправки регистрационных форм и сервис.

---

## Файлы для правок (каноничные URL — только индексы в папках)

| Язык | URL | Файл |
|------|-----|--------|
| RU | `/` (корень) и `/ru/` | `index.html`, `ru/index.html` |
| EN | `/en/` | `en/index.html` |
| UK | `/uk/` | `uk/index.html` |
| PL | `/pl/` | `pl/index.html` |
| CZ | `/cs/` | `cs/index.html` |

**Удалены как неиспользуемые:** `ua.html`, `pl.html`, `cz.html`, `en.html`, `ru.html` — контент только по путям `/uk/`, `/pl/`, `/cs/`, `/en/`, `/ru/`.

Во всех файлах структура секций совпадает; меняются только тексты по языку.

## 1. HERO SECTION (`#hero`)

**Что менять:**
- **H1** — заменить на новый заголовок из ТЗ (Neuro.Educatimo: «Меняем „веру в результат“…» и т.д. по языкам).
- **Подзаголовок** (один `<p>`) — заменить на новый длинный подзаголовок из ТЗ.
- **Кнопка CTA** — текст ссылки/кнопки: «Внедрить доказательный подход» (и эквиваленты EN/PL/CS/UK).

**Вёрстка:**  
Подзаголовок длинный — проверить переносы и отступы. При необходимости в `style.css`: чуть уменьшить `font-size` или `line-height` в `#hero p`, чтобы блок оставался компактным и симметричным. Кнопку не растягивать; при длинном тексте (например PL) — допускается `text-align: center` и при необходимости `padding` по горизонтали.

## 2. PROBLEMS — блок «Боли клиента» (`#problem-solution`)

**Что менять:**
- Заголовок секции (`.section-header h2`) и описание (`.section-header p`) — в ТЗ явно не заданы; оставить текущие или заменить на нейтральный вариант вроде «Боли клиента» / «Customer pains» по языкам (на усмотрение).
- **Три текущие карточки** (`.problem-item`) — заменить заголовки и тексты на три из ТЗ:
  1. Клиенты уходят «в никуда»
  2. Зависимость от педагогов
  3. Упущенная прибыль
- **Добавить четвёртую карточку** (новая `.problem-item`): «Образовательный кризис (PISA)» с текстом из ТЗ.

**Вёрстка:**  
Сейчас сетка: `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr))` — четыре карточки будут выстраиваться в 2x2 или в ряд в зависимости от ширины. Проверить, что высоты карточек выровнены (например `align-items: start` или фиксированная `min-height`), чтобы блок выглядел аккуратно. Длинные заголовки/тексты не должны ломать сетку.

## 4. OPPORTUNITIES — блок «Возможности» (`#features`)

**Что менять:**

**Вёрстка:**  
Тексты длиннее текущих — проверить, что карточки не «разъезжаются» по высоте. При необходимости задать единую `min-height` для `.feature-item` или ограничить число строк с помощью `line-clamp` / фиксированной высоты, чтобы блок выглядел симметрично.

## 9. Стили (style.css) — точечные правки

- **Hero:** при необходимости — `#hero p { font-size: 0.95em; }` или `max-width` для подзаголовка.
- **Кнопки:** при длинном CTA — проверить `.btn` на мобильных; при необходимости `padding-left/right`, `font-size`, `white-space`.
- **problem-solution-grid:** при 4 карточках — при необходимости `min-height` у `.problem-item` для выравнивания.
- **feature-item:** при необходимости `min-height` или ограничение высоты текста для симметрии.

Никаких глобальных изменений верстки; только то, что нужно для аккуратного отображения новых текстов.
